Back-end

Você deverá projetar e construir um banco de dados MySQL e uma API REST em PHP para manipular esses dados.

IMPORTANTE:

  • Sua API deve seguir a arquitetura REST
  • Sua API deve retornar as respostas em JSON.
  • Não utilize Functions ou Procedures do MySQL, implemente a lógica em seu código PHP.

Você pode utilizar qualquer framework de PHP para a construção da API, assim como pode também optar por não utilizar nenhum. Se optar por utilizar um framework para a construção da API, você ganhará pontos-extra se optar pelo Laminas Framework.

Você pode utilizar qualquer ORM para comunicação com o banco de dados, assim como pode também optar por não utilizar nenhum. Se optar por utilizar um ORM, você ganhará pontos-extra se optar pelo Doctrine.

Entidades e requisitos do domínio

Abaixo estão listadas as entidades existentes no domínio e os requisitos para cada uma delas. Se você não obedecer aos requisitos, você será desclassificado.

Turma

Cada turma possui um nome e uma série. Uma turma pode possuir zero ou mais alunos. Não é permitido existir duas turmas com o mesmo nome.

Aluno

Cada aluno possui um nome, email e data de nascimento. Um aluno deve, a todo momento, pertencer a uma, e apenas uma turma. Ou seja, um aluno não deve existir sem uma turma. Não é permitido existir dois alunos com o mesmo email.

Requisitos da API

Os requisitos são divididos em Requisitos Obrigatórios (RO) e Requisitos Extra (RE). Se você não fizer algum dos requisitos obrigatórios, será automaticamente desclassificado. Os Requisitos Extra garantirão pontos-extra na sua avaliação. O conteúdo do corpo das requisições e respostas da sua API ficam a seu critério, assim como o formato dos endpoints, mas tome cuidado para não fugir da arquitetura REST.

Requisitos Obrigatórios (RO)

RO1 - Cadastro de turmas

Sua API deve permitir cadastrar turmas.

RO2 - Atualização de turmas

Sua API deve permitir atualizar o nome das turmas.

RO3 - Listagem de turmas

Sua API deve permitir listar as turmas da escola. Os resultados devem ser ordenados por ordem alfabética do nome.

RO4 - Exclusão de turmas

Sua API deve permitir excluir turmas.

RO5 - Cadastro de alunos

Sua API deve permitir cadastrar alunos.

RO6 - Atualização de alunos

Sua API deve permitir atualizar o nome, email e data de nascimento dos alunos.

RO7 - Exclusão de alunos

Sua API deve permitir excluir alunos.

RO8 - Listagem de alunos da turma

Sua API deve permitir listar os alunos de uma turma específica. Os resultados devem ser ordenados por ordem alfabética do nome.

Requisitos Extra (RE)

RE1 - Autenticação (muitos pontos-extra)

A API deve ser protegida por autenticação, ou seja, não deve permitir que usuários não autenticados executem ações no sistema. O método de autenticação fica a seu critério, mas aqui estão alguns exemplos: chave de API ou login com usuário e senha.

RE2 - Transferência de alunos entre turmas

Permitir transferir um aluno de uma turma para outra.

RE3 - Filtrar turmas por nome

Na listagem de turmas, permitir filtrar os resultados por nome.

RE4 - Filtrar turmas por série

Na listagem de turmas, permitir filtrar os resultados por série.

RE5 - Filtrar alunos da turma por nome

Na listagem de alunos da turma, permitir filtrar os resultados por nome.

RE6 - Listagem de alunos da escola

Permitir listar todos os alunos da escola. Para cada aluno, deve-se indicar a qual turma o aluno pertence.

Entregas

Você deve entregar:

  • Link para o repositório da sua API;
  • Diagrama Entidade Relacionamento (DER) do seu banco de dados. Pode incluir como uma imagem em seu repositório.
  • Link para a sua API. Pode hospedá-la em qualquer lugar que desejar.
  • Instruções para testar cada um dos requisitos da API:
    • Endpoint;
    • Método HTTP;
    • Formato da payload;
    • Query strings;
  • Credenciais para autenticação caso tenha feito o Requisito Extra RE1.
  • Envie todas as informações para rh@kuau.com.br

O que será avaliado

  • Se todos os requisitos de domínio e os requisitos obrigatórios da API foram obedecidos.
  • Conhecimento sobre APIs REST
  • A estruturação de endpoints da sua API.
  • Métodos HTTP utilizados para cada ação na API.
    • As ações utilizam métodos HTTP apropriados para uma API REST?
  • Quantidade de requisitos extra da API que foram implementados.
  • Tratamento de erros da API
    • Os erros estão sendo tratados?
    • Quais códigos HTTP estão sendo utilizados? São adequados para o erro?
    • Informações adicionais nos erros.
  • A qualidade e a organização do seu código PHP.
  • Sua modelagem do banco de dados.
  • A Documentação da sua API
    • Listagem dos endpoints e métodos HTTP;
    • Listagem de erros retornados;
    • Formato da payload;
    • Query strings.

Front-end

Agora que você já criou o banco de dados e a API do sistema, você deverá criar um site para que irá comunicar com a API. Você deverá implementar os casos de uso descritos na seção Casos de Uso e enviar o seu projeto para um repositório púbico do GitHub.

A organização das páginas, a interface, as ferramentas e frameworks a serem utilizados ficam a seu critério. Pode-se utilizar qualquer framework CSS, ou até mesmo nenhum. Pode-se também utilizar qualquer linguagem de extensão de CSS, como Sass e Less. Caso opte por usar o Angular como framework web Javascript você irá ganhar pontos-extra.

IMPORTANTE: Não utilize o PHP para gerar o seu HTML.

Casos de uso

Os casos de uso estão separados em duas categorias: obrigatórios e extra. Se você não implementar os casos de uso obrigatórios, estará automaticamente desclassificado. Se você implementar os casos de uso extra irá ganhar “pontos extra”.

Casos de uso obrigatórios (CUO)

Seu site deve permitir realizar as ações descritas nos Requisitos Obrigatórios (RO) da API:

  • CUO1 - Cadastro de turmas;
  • CUO2 - Atualização de turmas;
    • Deve-se permitir atualizar o nome das turmas.
  • CUO3 - Listagem de turmas;
    • Os resultados devem ser ordenados por ordem alfabética do nome.
  • CUO4 - Exclusão de turmas;
  • CUO5 - Cadastro de alunos;
  • CUO6 - Atualização de alunos;
    • Deve-se permitir atualizar o nome, email e data de nascimento dos alunos.
  • CUO7 - Exclusão de alunos;
  • CUO8 - Listagem de alunos da turma.
    • Os resultados devem ser ordenados por ordem alfabética do nome.

Casos de uso extra (CUE)

Seu site deve permitir realizar as ações descritas nos Requisitos Extra (RE) da API:

  • CUE1 - Autenticação (muitos pontos-extra);
    • Apenas no caso do método de autenticação escolhido exigir login.
  • CUE2 - Transferência de alunos entre turmas;
  • CUE3 - Filtrar turmas por nome;
  • CUE4 - Filtrar turmas por série;
  • CUE5 - Filtrar alunos da turma por nome;
  • CUE6 - Listagem de alunos da escola.

Entregas

Você deve entregar:

  • Link para o repositório do GitHub;
  • Um link para um local onde seu site esteja hospedado para podermos testá-lo.
    • Você pode hospedar seu site gratuitamente no GitHub Pages, Heroku, dentre outros.
  • Credenciais para login caso necessário.
  • Envie todas as informações para rh@kuau.com.br

O que será avaliado

  • Conhecimento em HTML;
  • Conhecimento em CSS;
  • Conhecimento em Javascript/Typescript;
  • Conhecimento em Angular (caso tenha optado por utilizar);
  • Responsividade das páginas;
    • As páginas devem se adaptar para o uso em desktops, tablets e celulares.
  • Tratamento de erros de API recebidos;
  • Tratamento de respostas vazias recebidas da API.
  • Qualidade e criatividade na organização das páginas;
  • Qualidade e organização do código.
  • A quantidade de Casos de Uso Extra implementados.
